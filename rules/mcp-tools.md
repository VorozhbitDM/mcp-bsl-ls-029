# Справочник по инструментам MCP bsl-mcp

Этот файл описывает доступные инструменты MCP для работы с кодом 1С (BSL).

## Конфигурация сервера

- **Имя сервера в Kiro:** `bsl-mcp`
- **Python-пакет:** `mcp_bsl.server`
- **BSL Language Server:** `bsl-language-server-0.29.0-exec.jar` (путь задаётся через `BSL_JAR`)
- **Конфиг BSL LS:** `.bsl-language-server.json` (путь задаётся через `BSL_CONFIG`)
- **Память:** 4096 МБ (задаётся через `BSL_MEMORY_MB`)

> Имя сервера (`bsl-mcp`) — это только отображаемый идентификатор в Kiro.
> Реальный Python-пакет всегда `mcp_bsl` — его нельзя переименовать через конфиг.

## Инструменты

| Инструмент | Назначение | Когда использовать |
|---|---|---|
| **bsl_analyze** | Анализ BSL-кода — находит ошибки, предупреждения, подсказки (диагностики BSL Language Server) | После написания или изменения кода. **Лимит: 3 раза за цикл** |
| **bsl_format** | Форматирование BSL-файлов — выравнивает отступы, убирает лишние пробелы и пустые строки | Перед финальной проверкой кода |

## Параметры

Оба инструмента принимают один параметр:

- **srcDir** — путь к файлу `.bsl` или директории с BSL-файлами

Примеры:
- Файл: `CommonModules/ПМЩ_ОбщегоНазначения/Ext/Module.bsl`
- Папка: `CommonModules/ПМЩ_ОбщегоНазначения/Ext`

## Важные правила

1. **Сначала форматируй, потом анализируй** — запускай `bsl_format` перед `bsl_analyze`
2. **Ограничь bsl_analyze** — максимум 3 раза за цикл, затем продолжай работу
3. **Приоритет диагностик** — исправляй в первую очередь уровень Error, затем Warning
4. **Если не удаётся исправить ошибки линтера со 2-3 раза** — не пытайся снова, зафиксируй проблему и двигайся дальше
